Timestamp microservice

You can convert timestamp into natural date representation (e.g. January 1, 2016) and vice versa.



You can also do this through the API

API usage instructions:
    1) You need to pass a string parameter in URL({{ site_url}}_parameter_/), and it will check to see whether that string contains either a unix timestamp or a natural language date (e.g. January 1, 2016)
    2) If it does, it returns both the Unix timestamp and the natural language form of that date.
    3) If it does not contain a date or Unix timestamp, it returns null for those properties.

Example of API usage:

{{ site_url }}December 15, 2015/
{{ site_url }}1450137600/

Example output:

{ "unix": 1450137600, "natural": "December 15, 2015" }
